## Pipeline Flow

```
Image Upload → sharp (1024px) → parallel:
  ├─ tesseract.js OCR (ind+eng) → ocrText
  └─ AI Vision Captioning → caption
         └─ caption + ocrText → summary
                └─ caption + summary → embedding (1536d)
                       └─ store as multimodal chunk
```

## Multimodal Chunk Structure

```json
{
  "content": "Network topology diagram caption",
  "chunkType": "image",
  "ocrText": "Server 192.168.1.1 Gateway 10.0.0.1",
  "caption": "Network topology diagram showing web server connected to PostgreSQL cluster",
  "imageSummary": "This architecture consists of three layers: web server, application server, and database cluster",
  "imageUrl": "/uploads/1717881600000-diagram.png",
  "metadata": { "source": "image", "width": 1024, "height": 768 }
}
```

## Retrieval Flow

```
User Query → Embed → Vector Search (HNSW, threshold 0.30)
  → Returns text chunks + image chunks
  → buildMultimodalContext() formats:
      [Image: network-diagram.png]
      [Caption] Network topology diagram...
      [OCR Text] Server 192.168.1.1...
      [Summary] This architecture consists of...
  → Token budget enforced (8000 tokens)
  → Dedup near-duplicates
  → Feed to AI model
```

## Key Design Decisions

1. **tesseract.js over PaddleOCR** — No native deps, works in Docker, supports Indonesian + English
2. **Embedding from caption+summary** — Not raw OCR (OCR text can be noisy, caption is semantically rich)
3. **sharp for preprocessing** — Resize to 1024px max before sending to vision API (reduces token cost)
4. **Parallel OCR + Vision** — Both run simultaneously via Promise.all for speed
5. **Graceful degradation** — If OCR fails, empty string; if vision fails, generic caption
6. **Raw SQL for multimodal storage** — New columns not yet in Prisma schema (avoids full schema regen)
