## Visual Issues

1. **No brand/accent color** — Entire palette is monochromatic gray. Buttons, links, active states all look identical. Zero visual hierarchy.
2. **Chart colors are grayscale** — 5 chart colors are all gray steps. Charts are unreadable and cannot distinguish data series.
3. **Toaster hardcoded to light mode** — `background: "white"`, `border: "1px solid #e5e7eb"` breaks in dark mode.
4. **Markdown CSS hardcoded colors** — 12+ hex colors in globals.css don't respond to dark mode.
5. **Scrollbar hardcoded to light gray** — `#d1d5db` and `#9ca3af` not theme-aware.
6. **Dashboard is feature-heavy, not insight-heavy** — 8+ cards stacked vertically with no visual hierarchy or grouping.
7. **No status colors** — No success/warning/info semantic colors beyond destructive red.

## UX Issues

1. **Dashboard overload** — Shows StatCards, UsageChart, KBStats, CostSummary, RecentChats, TopDocuments, RetrievalAnalytics, EvaluationAnalytics, SystemHealth, PlanStatus — all stacked vertically. Users can't find what matters.
2. **No onboarding flow** — New users land on empty dashboard with no guidance.
3. **Upload is buried** — Hidden inside Knowledge Base collapsible section in sidebar.
4. **Settings scattered** — Workspace, Usage, Billing, Settings are 4 separate bottom nav items. Should be one Settings page with tabs.
5. **No feature discoverability** — Widgets, API, Playground, Prompts are hidden in collapsed sidebar sections.
6. **Empty states are unhelpful** — "No benchmark runs yet. Run: npx tsx scripts/run-rag-eval.ts" exposes CLI commands to end users.
7. **No loading coordination** — 6+ widgets independently show skeletons at different times. Looks glitchy on load.
8. **No dark mode toggle** — Users can only switch via OS preference.

## Information Architecture Issues

1. **Sidebar nesting too deep** — Knowledge Base > Documents/Chunks/Search/Sources/Upload. 5 items in one collapsible.
2. **Duplicate navigation paths** — "Documents" exists in sidebar AND as a top-level route `/documents`.
3. **Inconsistent grouping** — Analytics has Usage/Chat/Cost but no Retrieval or Evaluation (those are dashboard widgets).
4. **AI section misplaced** — Playground and Prompts under "AI" collapsible, separate from Chat.
5. **No clear primary action** — Dashboard doesn't guide users to their next action.

## Accessibility Issues

1. **No skip-to-content link** — Sidebar requires tab-through on every page load.
2. **No aria-labels on icon-only buttons** — User avatar button, refresh buttons.
3. **Color contrast** — Monochromatic gray palette may fail WCAG contrast ratios for text-on-background.
4. **No focus indicators** visible in the screenshots.
5. **Sidebar section labels use `text-[11px]`** — Below 12px minimum for readable text.

## Mobile Issues

1. **Only 2 breakpoints** — `sm` (640px) and `lg` (1024px). No `md` (768px) layout. Tablet gets mobile layout.
2. **Mobile sidebar width (280px) ≠ Desktop (260px)** — Inconsistent.
3. **Analytics tables overflow** — Worst Performers, Benchmark History rows overflow on screens < 380px.
4. **Dashboard chart height hardcoded** — Fixed pixel height may overflow on small screens.
5. **No bottom navigation on mobile** — Only hamburger menu. Users must open sidebar for every navigation.
